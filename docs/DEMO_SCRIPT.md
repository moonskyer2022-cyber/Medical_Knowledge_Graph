# 3 分钟演示脚本

## 0:00–0:30：启动与定位

```powershell
powershell -ExecutionPolicy Bypass -File scripts/demo.ps1
```

介绍：这是一个基于 Neo4j 的医药知识图谱 Demo，提供疾病推荐、药品查询、联用风险、引用溯源和安全护栏。

## 0:30–1:10：疾病方案推荐

在推荐页面输入“高血压”，展示：

- 一线/二线治疗方案；
- 药品列表和证据等级；
- 指南来源和图谱关系。

强调系统给出的是可追溯的知识检索结果，而不是无来源的文本回答。

## 1:10–1:50：处方风险检查

输入“阿司匹林”和“氯吡格雷”，展示：

- 相互作用严重程度；
- 风险描述与建议；
- 图谱中的药品关系。

说明系统可以发现已登记的联用风险，但不能覆盖所有临床因素。

## 1:50–2:25：Graph-RAG 与安全护栏

在问答页面依次演示：

1. “阿司匹林和氯吡格雷能一起用吗？”——展示结构化回答和引用。
2. “二甲双胍剂量需要加量吗？”——展示 `review_required`。
3. “服药后胸痛怎么办？”——展示 `blocked` 和急诊提示。

## 2:25–2:50：来源治理

展示来源列表、来源审核状态和审核历史，说明：

- 元数据核验不等同于临床内容审核；
- 临床审核需要指定角色；
- 审核决定会留下可追溯记录。

## 2:50–3:00：工程质量

展示 README、GitHub Actions 和测试结果：

- 单元测试与真实 Neo4j E2E；
- Docker 可复现启动；
- 认证、审计和来源登记。
