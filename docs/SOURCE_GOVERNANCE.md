# 医学来源治理与审核状态

`data/source_registry.csv` 是图谱中指南和共识来源的唯一登记表。每个来源包含稳定 ID、发布方、版本、发布日期、公开链接、审核日期和审核状态；`scripts/clean.py` 会拒绝导入未登记的来源名称。

## 审核状态

- `metadata_verified`：已在公开的发布方、学会或期刊页面核验标题、版本或发布记录与链接。此状态**不表示**图谱中的每条药物关系、剂量、禁忌或相互作用已经过医学专家逐条审核。
- `unverified_demo`：仅有演示数据中的来源名称；尚未核验可公开访问的正式版本和链接。此类来源不能作为对外临床证据声明。

## 当前审核结果

截至 2026-07-11，`中国高血压防治指南`、`中国2型糖尿病防治指南`、`中国血脂管理指南（2023年）` 和 `中国胃食管反流病多学科诊疗共识2022（一）` 已完成公开元数据核验。其余登记项仍为 `unverified_demo`，在获得医学审核人、正式版本和公开链接前保持该状态。

## 发布门槛

任何来源从 `unverified_demo` 提升到 `metadata_verified` 前，必须更新来源登记表并提供可审计链接。若要作为临床决策支持的证据，还必须完成领域专家的内容审查并记录审查结论、审查人和日期。

## 审核操作

审核员可通过管理员 API 提交 `metadata` 或 `clinical_content` 审核；系统会校验角色、审核结论和已批准记录的证据 URL，并将每次决定追加到 `data/source_reviews.csv`。接口和 CLI 共用同一套持久化逻辑，避免审核状态分叉。
