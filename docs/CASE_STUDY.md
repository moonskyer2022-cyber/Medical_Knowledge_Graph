# 端到端业务案例：合并症用药风险评估

## 场景

演示对象为一名同时存在高血压和 2 型糖尿病的成人患者。目标不是自动开方，而是展示系统如何把疾病、方案、药品、剂量、禁忌和相互作用串成一条可解释的辅助分析链路。

## 输入

- 疾病：高血压、2 型糖尿病
- 当前用药：阿司匹林、氯吡格雷（用于演示联用风险）
- 关注条件：肾功能不全或妊娠相关禁忌

## 系统处理链路

1. 从疾病名称解析 `Disease` 节点。
2. 通过 `TARGETS` 和 `INCLUDES` 关系检索治疗方案与药品。
3. 对合并后的药品集合执行 `INTERACTS_WITH` 和 ATC 重复用药检查。
4. 根据条件查询 `CONTRAINDICATED_FOR` 关系。
5. 生成结构化结果、Graph-RAG 引用、审核状态和安全提示。
6. 对剂量、停换药等个体化问题标记 `review_required`，不自动替代临床判断。

## 预期输出

```text
疾病方案：高血压一线方案、2 型糖尿病基础治疗
风险项：双抗联用可能增加出血风险
来源：指南标题、版本、来源 ID、审核状态
安全提示：结果仅供辅助参考，需要医生或药师复核
```

## 业务价值

- 将多个疾病的方案放在同一张图中分析，避免只看单一疾病。
- 用关系和证据来源解释风险，而不是只输出一个黑盒结论。
- 通过安全护栏和人工复核状态控制医疗场景风险。

## 演示边界

本案例使用演示数据，不包含真实患者身份信息，也不应被用于诊断、处方或急救决策。
