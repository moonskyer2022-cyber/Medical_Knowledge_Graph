# 产品与工程演进路线

## 已完成：Demo 基础

- Neo4j 图谱建模与 CSV 清洗导入
- FastAPI 业务接口和 Web 展示
- Graph-RAG 结构化引用
- 医疗安全护栏
- Docker Demo、CI、真实 Neo4j E2E
- Bearer Token、角色控制、审计和来源审核工作流

## 下一阶段：可交付 PoC

- 增加完整管理页面：来源审核、审计查询、审核历史时间线
- 将演示账号替换为数据库或企业 SSO
- 集中化审计、日志轮换、告警和权限审计
- 为关键查询增加性能基线和容量测试
- 补充更多经过医学专家审核的指南与说明书数据

## 后续阶段：企业级临床辅助平台

- 接入 FHIR/HIS/EMR，并完成脱敏和租户隔离
- 增加向量检索和 LLM，但限制模型只能基于已审核证据生成
- 建立模型、知识库和规则的版本回滚机制
- 增加人工确认、反馈纠错和临床效果评估
- 完成隐私、数据授权、医疗软件合规与安全评审
